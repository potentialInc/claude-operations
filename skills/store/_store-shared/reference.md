# Store Shared Reference

Common rules and conventions shared by all store skills (`store-*`).

---

## Pre-flight: Gitignore Output Directory

All store skills generate output files into `store-prep/` at the project root.
These artifacts are for PM review only and must NOT be committed to git.

**Every store skill MUST execute this pre-flight before creating any files:**

1. Check if `.gitignore` exists in the project root
2. If it exists, read it and check whether `store-prep/` is already listed (match `store-prep/` or `store-prep` on its own line)
3. If not listed, append the following block to `.gitignore`:

```
# Store submission artifacts (generated by store-* skills)
store-prep/
```

4. If `.gitignore` does not exist, create it with the block above

**This step is non-negotiable** — skip it only if `store-prep/` is already in `.gitignore`.
