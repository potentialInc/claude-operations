## 카테고리별 패턴 목록

### 카테고리: 인증/로그인

---

### Pattern: 세션 만료/자격증명 오류 반복 노출
- **빈도**: 5회 (높음)
- **증상**: 사용자가 정상 사용 중 자격증명 오류 팝업이 반복 노출됨. 페이지를 새로고침하면 해결되지만 빈번히 재발생
- **근본 원인**: 서버 세션 제한 또는 토큰 갱신 실패 시 클라이언트에서 적절한 핸들링 없이 에러를 그대로 노출
- **영향 화면**: 대시보드, 드롭다운 메뉴, 홈 화면
- **예방 스펙**: "[클라이언트] 세션 만료(401/403) 응답 수신 시 자동으로 토큰 갱신을 시도하고, 갱신 실패 시 로그인 페이지로 리다이렉트하며 사용자에게 '세션이 만료되었습니다' 메시지를 1회만 표시한다. 동일 에러를 중복 노출하지 않도록 에러 상태를 전역으로 관리한다."
- **사례**: [#1252, #1251, #1244, #1240, #1576]

---

### Pattern: 동일 계정 다중 기기 로그인 처리 오류
- **빈도**: 3회 (중간)
- **증상**: 이미 다른 기기에서 로그인 중임에도 동일 계정으로 재로그인 가능하거나, 반대로 정상 사용 중 "다른 기기에서 이용 중" 메시지가 반복 표시됨
- **근본 원인**: 단일 기기 로그인 강제 로직의 세션 무효화 처리 미흡 또는 락 해제 타이밍 불일치
- **영향 화면**: 로그인 화면, 홈 화면
- **예방 스펙**: "[서버] 로그인 시 기존 세션을 즉시 무효화하고 새 세션을 발급한다. [클라이언트] 다중 기기 로그인 감지 시 명확한 안내 후 기존 세션을 강제 종료하는 옵션을 제공한다. 로그아웃 시 서버 세션을 완전히 삭제하여 재로그인 시 오탐이 발생하지 않도록 한다."
- **사례**: [#2225, #2329, #1575]

### Pattern: 인증 후 잘못된 화면으로 리다이렉트
- **빈도**: 4회 (중간)
- **증상**: 로그인 완료 후 대시보드가 아닌 비밀번호 재설정 완료 화면, 로그인 페이지 등 잘못된 화면으로 이동. 비인증 사용자가 홈 화면에 접근 가능
- **근본 원인**: 인증 상태 체크 및 라우팅 가드 로직 미적용 또는 네비게이션 스택 관리 오류
- **영향 화면**: 로그인 화면, 회원가입 완료 화면, 홈 화면
- **예방 스펙**: "[클라이언트] 모든 라우트에 인증 가드를 적용한다. 미인증 사용자의 보호된 라우트 접근 시 로그인 페이지로 리다이렉트한다. 인증된 사용자가 로그인/온보딩 페이지 접근 시 대시보드/홈으로 리다이렉트한다. 회원가입 완료 후 네비게이션 스택을 초기화하여 뒤로가기로 인증 전 화면에 접근할 수 없도록 한다."
- **사례**: [#2226, #2234, #2928, #2894]

### Pattern: 비밀번호 유효성 검사 미흡
- **빈도**: 3회 (중간)
- **증상**: 규칙(8-12자, 문자+숫자+특수문자)에 맞지 않는 비밀번호로 회원가입 가능. 비밀번호 변경 성공 후 404 에러 표시. 잘못된 비밀번호 입력 시 에러 메시지 미흡
- **근본 원인**: 클라이언트 측 유효성 검사 누락 또는 서버 응답 후 성공/실패 분기 처리 오류
- **영향 화면**: 회원가입, 비밀번호 변경, 로그인
- **예방 스펙**: "[클라이언트] 비밀번호 입력 필드에 실시간 유효성 검사를 적용하고, 규칙 미충족 시 구체적인 안내 메시지를 표시한다. [서버] 비밀번호 변경 API는 성공(200) 시 성공 응답을, 실패 시 적절한 에러 코드를 반환한다. [클라이언트] 잘못된 비밀번호 입력 시 '아이디 또는 비밀번호가 일치하지 않습니다' 메시지를 표시한다."
- **사례**: [#2273, #1272, #2932, #2235, #2240]

### Pattern: 소셜 로그인 실패
- **빈도**: 2회 (낮음)
- **증상**: Apple 계정 로그인이 동작하지 않음. 인증 프로세스 마지막 단계에서 서버 오류 발생
- **근본 원인**: 소셜 로그인 콜백 처리 또는 서버 측 토큰 검증 로직 오류
- **영향 화면**: 로그인 화면
- **예방 스펙**: "[서버] 소셜 로그인(Apple, Google 등) 콜백 처리 시 토큰 검증 실패에 대한 명확한 에러 응답을 반환한다. [클라이언트] 소셜 로그인 실패 시 사용자에게 재시도 안내 메시지를 표시하고 일반 로그인 옵션을 제공한다."
- **사례**: [#2243, #2935]

---

### Pattern: 온보딩 화면 반복 노출
- **빈도**: 3회 (중간)
- **증상**: 이미 온보딩을 완료한 사용자에게 앱 시작 시 또는 로그아웃 후 재로그인 시 온보딩 화면이 반복 표시됨
- **빈도**: 3회 (중간)
- **근본 원인**: 온보딩 완료 상태를 로컬 스토리지에 저장하지 않거나 초기화 시 상태 확인 로직 누락
- **영향 화면**: 온보딩, 스플래시 화면
- **예방 스펙**: "[클라이언트] 온보딩 완료 여부를 로컬 스토리지(AsyncStorage/SecureStore)에 저장하고, 앱 시작 시 이 값을 확인하여 온보딩 건너뛰기를 결정한다. 로그아웃 시 온보딩 완료 플래그는 초기화하지 않는다."
- **사례**: [#2946, #2954, #2887]

### 카테고리: 카메라/권한

---

### Pattern: 카메라 권한 요청 타이밍 및 처리 오류
- **빈도**: 4회 (중간)
- **증상**: 앱 시작 직후(스플래시 이후) 카메라 권한 요청. 이미 권한을 허용한 사용자에게 재로그인 시 권한 화면 반복 표시. '권한 요청' 버튼 동작 안 함. 홈 화면에서 카메라가 동작하지 않음
- **근본 원인**: 권한 요청 타이밍이 적절하지 않고, 권한 부여 상태를 로컬에 캐싱하지 않아 매번 재확인
- **영향 화면**: 스플래시, 온보딩, 홈 화면, 매칭 화면
- **예방 스펙**: "[클라이언트] 카메라 권한 요청은 해당 기능이 실제로 필요한 시점(온보딩 로그인 이후 또는 매칭 시작 전)에 요청한다. 권한 허용 여부를 로컬에 캐싱하고 재로그인 시 재요청하지 않는다. 권한 거부 시 설정으로 이동할 수 있는 안내 UI를 제공한다."
- **사례**: [#1967, #1573, #2199, #1568]

### 카테고리: 실시간 데이터

---

### Pattern: 실시간 메시지/알림 미수신
- **빈도**: 5회 (높음)
- **증상**: DM 메시지가 실시간으로 표시되지 않음. 특정 페이지에서만 채팅 알림 수신. 다른 페이지에서 메시지 수신 시 알림 없음. 결제 알림 미수신. 선물 수신 알림 미수신
- **근본 원인**: 웹소켓/실시간 연결이 특정 화면에만 유지되고 전역 관리되지 않음. 포그라운드/백그라운드 전환 시 연결 재수립 로직 미흡
- **영향 화면**: DM 화면, 홈 화면, 주문 상세, 결제 화면
- **예방 스펙**: "[클라이언트] 웹소켓/실시간 연결은 앱 레벨에서 전역으로 관리하여 특정 화면에 종속되지 않게 한다. 앱이 포그라운드로 돌아올 때 연결 상태를 확인하고 끊어진 경우 재연결한다. 미읽은 메시지/알림은 배지 형태로 화면 전환과 무관하게 유지한다."
- **사례**: [#2263, #2239, #1594, #2931, #1262]

---

### Pattern: 온라인/오프라인 상태 표시 오류
- **빈도**: 1회 (낮음)
- **증상**: 상대방이 실제로 접속 중임에도 DM 창에서 오프라인으로 표시됨
- **근본 원인**: 사용자 상태 업데이트가 실시간으로 구독되지 않거나 초기 로드 시 상태를 가져오지 않음
- **영향 화면**: DM 화면
- **예방 스펙**: "[클라이언트] DM 화면 진입 시 상대방의 온라인 상태를 실시간 구독하고, 상태 변경 시 즉시 UI를 업데이트한다. [서버] 사용자 접속/이탈 이벤트를 웹소켓으로 브로드캐스트한다."
- **사례**: [#1563]

### 카테고리: 네비게이션/라우팅

---

### Pattern: 탭/화면 전환 후 잘못된 화면 표시
- **빈도**: 3회 (중간)
- **증상**: 소셜 탭으로 돌아왔을 때 메인 뷰 대신 설정 메뉴가 표시됨. 탭별 헤더 네비게이션이 다르게 표시됨. 이전 페이지로 돌아갈 때 이전 데이터가 유지되며 충돌 발생
- **근본 원인**: 탭 네비게이션 스택이 초기화되지 않거나 화면 포커스 시 상태를 재설정하지 않음
- **영향 화면**: 소셜 탭, 홈 탭, 주문 편집 화면
- **예방 스펙**: "[클라이언트] 각 탭의 네비게이션 스택을 독립적으로 관리하고, 탭 전환 시 스택을 루트로 초기화하는 옵션을 제공한다. 화면 포커스 이벤트(useFocusEffect)를 활용하여 화면 재진입 시 필요한 상태를 초기화한다."
- **사례**: [#1599, #2345, #1255]

---

### Pattern: 딥링크/알림 클릭 후 잘못된 화면 이동
- **빈도**: 2회 (낮음)
- **증상**: 알림 클릭 시 해당 콘텐츠로 이동하지 않음. 디자인 파일 추가 알림 클릭 시 리뷰 화면으로 직접 이동하지 않음
- **근본 원인**: 딥링크 처리 로직 미구현 또는 알림 페이로드와 라우팅 연결 누락
- **영향 화면**: 알림 목록, 디자인 리뷰 화면
- **예방 스펙**: "[클라이언트] 푸시 알림 및 인앱 알림 클릭 시 알림 타입에 맞는 화면으로 직접 이동하는 딥링크를 구현한다. 알림 페이로드에 target_screen과 target_id를 포함하여 정확한 라우팅을 보장한다."
- **사례**: [#1268, #2346]

### 카테고리: 폼/입력

---

### Pattern: 키보드 UI 가림 현상
- **빈도**: 5회 (높음)
- **증상**: 소프트 키보드 표시 시 입력 필드, 버튼, 네비게이션 바 등이 가려짐. 텍스트가 다음 줄로 넘어갈 때 커서가 키보드에 가려짐. 시스템 네비게이션 바가 모달/탭바를 가림
- **근본 원인**: KeyboardAvoidingView 또는 ScrollView 미적용. SafeAreaInsets 미처리. 동적 스크롤 로직 누락
- **영향 화면**: 채팅 입력, 회원가입 폼, 온보딩, 모달, 하단 탭바
- **예방 스펙**: "[클라이언트] 입력 필드가 있는 모든 화면에 KeyboardAvoidingView를 적용한다. react-native-safe-area-context의 useSafeAreaInsets를 사용하여 시스템 네비게이션 바 영역에 padding을 적용한다. 멀티라인 입력 시 커서 위치를 추적하여 화면을 자동 스크롤한다."
- **사례**: [#2039, #2242, #2247, #2248, #1571]

---

### Pattern: 폼 유효성 검사 미흡 또는 Continue 버튼 비정상 활성화
- **빈도**: 3회 (중간)
- **증상**: 필수 항목(배송 주소) 미입력 시 Continue 버튼 활성화. 부분 선택만으로 Continue 버튼 비활성화(전체 선택 필요). 회원가입 Continue 버튼 동작 안 함
- **근본 원인**: 폼 유효성 상태 관리 로직 오류 또는 일부 필드만 검사하는 버그
- **영향 화면**: 주문 생성 폼, 회원가입 폼, 서비스 선택 폼
- **예방 스펙**: "[클라이언트] Continue/Submit 버튼은 모든 필수 필드가 유효한 경우에만 활성화한다. 각 필드의 유효성 상태를 개별적으로 추적하고 AND 조건으로 버튼 활성화 여부를 결정한다. 선택형 필드는 '최소 1개 선택' 등 조건을 명확히 정의하여 구현한다."
- **사례**: [#1238, #1250, #1247]

---

### Pattern: 텍스트 에디터 서식(줄바꿈, 불릿) 미반영
- **빈도**: 2회 (낮음)
- **증상**: FAQ 텍스트 박스에서 불릿/번호 서식을 적용해도 실제 페이지에 반영되지 않음. 줄바꿈이 동작하지 않음
- **근본 원인**: Rich text editor의 HTML 출력을 렌더링 측에서 파싱하지 않거나 plain text로 저장
- **영향 화면**: FAQ 관리 페이지, 커뮤니티 포스트
- **예방 스펙**: "[서버] Rich text 콘텐츠를 HTML 또는 Markdown 형식으로 저장한다. [클라이언트] 저장된 Rich text를 렌더링 시 dangerouslySetInnerHTML 또는 Markdown 파서를 사용하여 서식을 유지한다."
- **사례**: [#1256, #1117]

### 카테고리: UI/레이아웃

---

### Pattern: 프로필 이미지 화면 간 불일치
- **빈도**: 3회 (중간)
- **증상**: 홈, 샵, 마이프로필 화면에서 프로필 이미지가 각각 다르게 표시됨. 프로필 미설정 시 더미 이미지가 표시되어야 하는데 기본 아바타가 표시됨
- **근본 원인**: 각 화면이 별도로 프로필 데이터를 로드하고 공유 상태를 사용하지 않음. 프로필 업데이트 후 전역 상태 미갱신
- **영향 화면**: 홈 화면, 샵 화면, 마이프로필 화면
- **예방 스펙**: "[클라이언트] 사용자 프로필 정보(이미지 포함)를 전역 상태(Context/Redux/Zustand)로 관리한다. 프로필 업데이트 시 전역 상태를 즉시 갱신하여 모든 화면에 일관된 데이터를 표시한다. 프로필 이미지 미설정 시 표시할 기본 이미지를 전역으로 통일한다."
- **사례**: [#1603, #1595, #1597]

### Pattern: 헤더/네비게이션 UI 일관성 부재
- **빈도**: 2회 (낮음)
- **증상**: 홈 화면과 샵 화면의 헤더 프로필 UI가 다름. 탭별로 헤더 네비게이션 컴포넌트가 다르게 표시됨
- **근본 원인**: 공통 컴포넌트를 사용하지 않고 각 화면에서 별도 구현
- **영향 화면**: 홈, 샵, 소셜 탭 헤더
- **예방 스펙**: "[클라이언트] 헤더, 네비게이션바 등 공통 UI 컴포넌트를 단일 공유 컴포넌트로 구현하고 모든 화면에서 재사용한다. 디자인 시스템에 공통 컴포넌트 스펙을 정의하고 화면별 커스터마이징을 props로 제어한다."
- **사례**: [#1562, #2345]

### Pattern: 스플래시/온보딩 이미지 잘림
- **빈도**: 1회 (낮음)
- **증상**: 스플래시 화면 이미지가 잘려서 표시됨. 온보딩 프로필 완성 페이지가 스크롤 불가
- **근본 원인**: 화면 크기에 대한 반응형 레이아웃 미적용
- **영향 화면**: 스플래시 화면, 온보딩 화면
- **예방 스펙**: "[클라이언트] 이미지는 resizeMode를 'contain' 또는 'cover'로 명시적으로 설정한다. 콘텐츠가 화면을 넘칠 수 있는 온보딩 화면에는 ScrollView를 기본 적용한다."
- **사례**: [#2038, #2906]

### 카테고리: 관리자 대시보드

---

### Pattern: 관리자 채팅 HTML 태그 노출 및 링크 비동작
- **빈도**: 1회 (낮음)
- **증상**: 관리자 채팅 미리보기에 HTML 태그가 그대로 노출됨. 링크 클릭 불가
- **근본 원인**: 채팅 메시지를 plain text로 렌더링하여 HTML 태그가 escape되지 않고 노출. 링크 컴포넌트 미사용
- **영향 화면**: 관리자 채팅 목록/미리보기
- **예방 스펙**: "[클라이언트] 채팅 메시지 렌더링 시 HTML을 파싱하여 서식을 적용하거나, 링크는 클릭 가능한 컴포넌트로 렌더링한다. 메시지 미리보기는 HTML 태그를 제거한 plain text로 표시한다."
- **사례**: [#1278]

---

### Pattern: 관리자 대시보드 통계/리포트 미갱신
- **빈도**: 2회 (낮음)
- **증상**: 리포트 페이지의 수치가 변경되지 않음. 결제 발생 시 관리자 패널 수입에 반영 안 됨
- **근본 원인**: 통계 데이터를 실시간 또는 주기적으로 갱신하지 않고 초기 로드 데이터만 표시
- **영향 화면**: 관리자 리포트, 수입 대시보드
- **예방 스펙**: "[클라이언트] 통계/수입 데이터는 화면 포커스 시 재조회하거나 일정 주기(예: 30초)로 자동 갱신한다. [서버] 결제 완료 이벤트 발생 시 관련 통계를 즉시 갱신한다."
- **사례**: [#1257, #754]

### Pattern: 관리자 페이지네이션 상태 유지 실패
- **빈도**: 2회 (낮음)
- **증상**: 신청자 상태 변경 후 1페이지로 리다이렉트되지만 페이지네이션이 올바르지 않음. 다음 페이지 버튼 클릭 시 그룹 번호는 바뀌지만 주문 내용이 1페이지와 동일하게 유지됨
- **근본 원인**: 페이지 번호 상태 관리 오류 또는 API 호출 시 페이지 파라미터가 올바르게 전달되지 않음
- **영향 화면**: 관리자 신청자 목록, 주문 목록
- **예방 스펙**: "[클라이언트] 목록 항목 상태 변경(승인/거절 등) 후 현재 페이지를 유지하거나 명시적으로 1페이지로 이동한다. 페이지 번호를 URL 파라미터로 관리하여 상태 변경 후에도 올바른 페이지를 유지한다. [서버] 페이지네이션 API에서 offset/limit 파라미터를 정확히 처리한다."
- **사례**: [#969, #1248]

---

### Pattern: 환불 상태 관리 오류
- **빈도**: 1회 (낮음)
- **증상**: 승인/거절된 환불 요청에 대해 재요청이 가능함. 승인/거절 후 관리자에게 승인/거절 버튼이 사라짐
- **근본 원인**: 환불 상태에 따른 UI 조건부 렌더링 로직 오류 및 상태 전환 규칙 미구현
- **영향 화면**: 환불 관리 페이지
- **예방 스펙**: "[서버] 환불 요청은 상태(요청/승인/거절/완료)를 명확히 구분하고, 이미 처리된 요청에 대한 재요청을 거부한다. [클라이언트] 환불 상태에 따라 가능한 액션 버튼을 조건부로 표시하고, 처리 완료된 요청에는 상태 뱃지만 표시한다."
- **사례**: [#1276]

### 카테고리: 결제/인앱구매

---

### Pattern: 가격 계산 오류
- **빈도**: 2회 (낮음)
- **증상**: 소계와 최종 합계 금액이 일치하지 않음(중복 청구 의심). 결제 화면에서 가격 항목이 표시되지 않음
- **근본 원인**: 배송비 등 추가 항목이 소계에 포함된 후 다시 합산되는 이중 계산 로직 오류
- **영향 화면**: 결제 페이지, 주문 상세 페이지
- **예방 스펙**: "[서버] 결제 금액 계산 시 각 항목(상품, 배송, 추가 서비스)을 명확히 구분하고 최종 합계는 서버에서 계산하여 클라이언트에 전달한다. [클라이언트] 결제 전 화면에서 모든 가격 항목(상품별 금액, 배송비, 할인, 최종 합계)을 명시적으로 표시한다."
- **사례**: [#1242, #1241]

### Pattern: 게임 재화(커런시) 반영 오류
- **빈도**: 2회 (낮음)
- **증상**: 인앱 구매 후 게임 재화(골드/DIA)가 잔액에 반영되지 않음. 신규 사용자에게 초기 재화 미지급
- **근본 원인**: 결제 완료 후 서버 웹훅 처리 또는 클라이언트 상태 갱신 누락
- **영향 화면**: 샵 화면, 홈 화면, 재화 잔액 표시 영역
- **예방 스펙**: "[서버] 결제 완료 웹훅 수신 후 사용자 재화 잔액을 즉시 갱신한다. 신규 사용자 생성 시 초기 재화(골드 2,000, DIA 1,000)를 자동 지급한다. [클라이언트] 결제 완료 후 재화 잔액을 서버에서 재조회하여 화면을 갱신한다."
- **사례**: [#2991, #1574]

### 카테고리: 비디오 통화

---

### Pattern: 비디오 통화 연결 불안정
- **빈도**: 4회 (중간)
- **증상**: 비디오 통화 연결 실패. 연장 수락 후 연결 끊김. Next Match 시 카메라 화면 미표시(5회 이상 테스트 시 발생). 매칭 화면 이동 시 서버 오류
- **근본 원인**: WebRTC 연결 해제 및 재연결 로직 미흡. 통화 연장 시 세션 전환 처리 오류
- **영향 화면**: 비디오 통화 화면, 매칭 화면, 통화 연장 화면
- **예방 스펙**: "[클라이언트] 비디오 통화 연결 실패 시 자동 재연결(최대 3회)을 시도하고 사용자에게 연결 상태를 표시한다. 통화 연장/Next Match 시 기존 WebRTC 세션을 명시적으로 종료 후 새 세션을 시작한다. [서버] 매칭 API의 예외 처리를 강화하여 서버 오류 발생 시 클라이언트에 명확한 에러 응답을 반환한다."
- **사례**: [#3003, #3004, #2228, #2936]

### 카테고리: 데이터 표시

---

### Pattern: 조건부 항목 미선택 시에도 표시
- **빈도**: 3회 (중간)
- **증상**: 선택하지 않은 추가 서비스가 표시됨. 크라운 재료 옵션이 4개 이상 치아 범위 선택 시 표시되지 않음. 임플란트 케이스에 Stump shade가 표시됨
- **근본 원인**: 조건부 렌더링 로직 오류 또는 데이터 모델에서 선택/미선택 상태 구분 미흡
- **영향 화면**: 주문 생성 폼, 주문 상세 페이지
- **예방 스펙**: "[클라이언트] 추가 서비스/옵션 항목은 사용자가 명시적으로 선택한 경우에만 표시한다. 케이스 타입에 따라 표시 가능한 옵션 목록을 서버에서 정의하고 클라이언트는 이를 기반으로 조건부 렌더링한다."
- **사례**: [#1269, #1277, #1243]

### Pattern: 목록 내 정보 표시 오류
- **빈도**: 2회 (낮음)
- **증상**: 최근 업데이트 목록에 환자 이름과 주문 번호 형식 미준수. 주문 내 메시지 알림 아이콘 미표시
- **근본 원인**: 목록 아이템 렌더링 시 데이터 매핑 오류 또는 UI 컴포넌트 누락
- **영향 화면**: 최근 업데이트 목록, 주문 목록
- **예방 스펙**: "[클라이언트] 목록 아이템의 표시 형식(예: '환자명 (주문번호)')을 상수로 정의하고 모든 목록에서 일관되게 적용한다. 알림 아이콘 등 상태 표시 UI는 해당 상태가 존재하는 모든 목록 뷰에 포함한다."
- **사례**: [#1267, #1263]

### 카테고리: 검색/필터

---

### Pattern: 검색 기능 미구현 또는 오동작
- **빈도**: 2회 (낮음)
- **증상**: Invoice ID로 검색 불가. 필터 적용 후 올바른 결과가 표시되지 않음. 교재 기준 정렬이 올바르게 동작하지 않음
- **근본 원인**: 검색 파라미터가 API에 전달되지 않거나 서버 측 쿼리 로직 오류
- **영향 화면**: 주문 검색, 필터 화면, 교재 관리
- **예방 스펙**: "[서버] 검색 API는 Invoice ID를 포함한 주요 식별자 기반 검색을 지원한다. [클라이언트] 검색/필터 상태 변경 시 디바운스(300ms)를 적용하고 결과를 즉시 갱신한다. 검색 결과가 없을 때 빈 상태 UI를 표시한다."
- **사례**: [#1261, #1249, #1119]

### 카테고리: API/서버

---

### Pattern: API 500 에러 및 서버 예외 처리 미흡
- **빈도**: 2회 (낮음)
- **증상**: PUT /user/{id} 엔드포인트에서 최소 업데이트 요청에도 500 반환. 비밀번호 변경 성공 후 404 에러
- **근본 원인**: 컨트롤러에서 null 체크 또는 관계 처리 누락. 성공 응답 코드 오류
- **영향 화면**: 사용자 프로필 수정, 비밀번호 변경
- **예방 스펙**: "[서버] 모든 UPDATE 엔드포인트는 partial update를 지원하고 null/undefined 필드를 무시한다. 컨트롤러에서 DB 쿼리 전 입력값 유효성을 검사하고 적절한 HTTP 상태 코드(200, 400, 404, 500)를 반환한다. 전역 예외 핸들러를 구현하여 미처리 예외가 500으로 반환되지 않도록 한다."
- **사례**: [#1539, #1272]

### 카테고리: 커뮤니티/게시판

---

### Pattern: 커뮤니티 콘텐츠 표시 오류
- **빈도**: 3회 (낮음)
- **증상**: 커뮤니티 페이지 썸네일 이미지 동작 안 함. 게시글 캐러셀 이미지 확인 불가. 비로그인 사용자에게 게시글 클릭 불가(클릭 가능해야 함). 첨부파일 업데이트 실패
- **근본 원인**: 이미지 URL 처리 오류, 권한 체크 위치 오류(클릭 이벤트에서 인증 체크), 파일 업로드 핸들러 오류
- **영향 화면**: 커뮤니티 목록, 커뮤니티 상세
- **예방 스펙**: "[클라이언트] 이미지 렌더링 시 URL 유효성을 확인하고 실패 시 대체 이미지를 표시한다. 비로그인 사용자도 공개 게시글을 열람할 수 있도록 하고, 댓글/좋아요 등 인터랙션 시점에만 로그인을 요구한다. [서버] 파일 첨부 업데이트 API는 기존 파일 삭제 후 새 파일 저장을 트랜잭션으로 처리한다."
- **사례**: [#1012, #1138, #1011, #1352]
