# Permission Hints

## 역할 목록
| 역할 | 설명 | 출처 |
|------|------|------|
| Owner | 조직/빌링/멤버 관리 | 클라이언트 직접 명시 |
| Admin | 프로젝트/초대/피드백 관리 | 클라이언트 직접 명시 |
| Member | 화면 관리, 답변, 상태 변경 | 클라이언트 직접 명시 |
| Client | 자기 배정 프로젝트 보고 핀 코멘트 작성 | 클라이언트 직접 명시 |
| SuperAdmin | 전체 조직 조회/상태관리 | 클라이언트 직접 명시 |

## 명시적 권한 규칙
| 역할 | 가능 | 불가능 | 출처 |
|------|------|--------|------|
| Owner | 조직/빌링/멤버 관리 | — | 클라이언트 명시 |
| Admin | 프로젝트/초대/피드백 관리 | 빌링/조직 설정 (추정) | 클라이언트 명시 |
| Member | 화면 관리, 답변, 상태 변경 | 프로젝트 생성/삭제, 멤버 초대 (추정) | 클라이언트 명시 |
| Client | 배정 프로젝트 열람, 핀+코멘트 작성 | 화면 관리, 상태 변경 (추정) | 클라이언트 명시 |
| SuperAdmin | 전체 조직 조회, 상태 관리 | — | 클라이언트 명시 |

## 소유권 규칙
| 규칙 | 출처 |
|------|------|
| Client는 자기 배정된 프로젝트만 볼 수 있음 | 클라이언트 직접 명시 |
| 조직별 데이터 완전 분리 (멀티테넌시) | 클라이언트 직접 명시 |
| 코멘트 작성자만 자기 코멘트 수정/삭제 가능 (추정) | 일반적 패턴 |
