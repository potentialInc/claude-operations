## 4. Admin Dashboard

> 접근 권한: Admin 역할 전용. 모든 라우트는 `/admin` 프리픽스 하에 위치하며, 비인가 접근 시 로그인 페이지로 리다이렉트.

---

### 4.1 Page Architecture

**Access:** Admin role only (DB Value: `99`)

**Stack:** React, React Router, Zustand (상태관리), Tailwind CSS

#### Page Map

| Route | Page | 관리 대상 |
|-------|------|----------|
| `/admin` | 대시보드 (Dashboard Overview) | 전체 통계 |
| `/admin/users` | 회원 관리 | User |
| `/admin/users/:id` | 회원 상세 | User |
| `/admin/shops` | 장착점 관리 | Shop |
| `/admin/shops/:id` | 장착점 상세 | Shop |
| `/admin/products` | 상품 관리 | Product |
| `/admin/products/:id` | 상품 상세 | Product |
| `/admin/orders` | 주문 관리 | Order |
| `/admin/orders/:id` | 주문 상세 | Order |
| `/admin/settlements` | 정산 관리 | Settlement |

### 4.2 Feature List by Route

#### `/admin` — Dashboard Overview

**Statistics Cards**

| 카드 | 측정값 | 비교 기준 |
|------|--------|----------|
| 전체 회원 수 | 누적 가입자 수 | 전일 대비 증감% |
| 신규 주문 수 | 기간 내 주문 건수 | 전 기간 대비 증감% |
| 총 매출액 | 기간 내 결제 완료 금액 (원) | 전 기간 대비 증감% |
| 장착점 수 | 승인된 활성 장착점 수 | 전 기간 대비 증감% |
| 미처리 문의 수 | 답변 대기 상태 문의 수 | 실시간 |
| 대기 중 정산 건수 | 정산 미처리 건수 | 실시간 |

**Period Filter**
- 오늘 / 최근 7일 / 최근 30일 / 사용자 지정 날짜 범위 (date picker)
- 선택한 기간에 따라 모든 통계카드 및 차트 동적 업데이트

**Charts**
- 일별 주문 건수 추이 (라인 차트, 기간 필터 연동)
- 일별 매출 금액 추이 (바 차트, 기간 필터 연동)
- 타이어 브랜드별 판매 비율 (도넛 차트)
- 장착점 배송 vs 자택 배송 비율 (파이 차트)

**Recent Activity**
- 최근 주문 목록 (최근 10건): 주문번호, 회원명, 상품명, 금액, 상태, 주문시각
- 최근 가입 회원 (최근 5건): 이름, 이메일, 가입일시
- 최근 1:1 문의 (최근 5건): 문의 제목, 작성자, 작성일시, 상태

**상태 화면**
- Loading: 통계카드 및 차트 영역에 skeleton UI 표시
- Empty: 데이터 없는 기간 선택 시 "해당 기간에 데이터가 없습니다" 안내
- Error: "대시보드 데이터를 불러올 수 없습니다. 새로고침 버튼" 표시

**에러 핸들링**
- 네트워크 에러: 오프라인/타임아웃 시 "인터넷 연결을 확인해주세요" 토스트 + 재시도 버튼
- 서버 에러 (500/503): "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요" + Retry 버튼
- 권한 에러: 세션 만료 시 로그인 페이지 리다이렉트

⚠️ **알려진 위험 (버그 패턴 기반)**
| # | 패턴 | 빈도 | 예방 스펙 |
|---|------|------|----------|
| 1 | 세션 만료/자격증명 오류 반복 노출 | 높음 (5회) | 401/403 수신 시 토큰 자동 갱신 시도, 갱신 실패 시 로그인 리다이렉트, 에러 메시지 1회만 표시 |
| 2 | 결제 완료 후 UI 미반영 | 높음 (7회) | 웹훅 수신 후 즉시 대시보드 통계 갱신 반영, 폴링 또는 WebSocket으로 실시간 업데이트 |

#### `/admin/users` — 회원 관리

**Top Area**
- **Search**: 이름, 이메일, 전화번호 키워드 검색 (debounce 300ms)
- **Filters**:
  - 회원 상태: 전체 / 활성(Active) / 정지(Suspended) / 탈퇴(Withdrawn)
  - 가입일 범위: date range picker (시작일 ~ 종료일)
  - 로그인 유형: 전체 / 이메일 / 소셜
- **Create button**: "+ 회원 추가" → 회원 생성 Modal 오픈
- **Bulk Action dropdown**: 일괄 활성화 / 일괄 정지 / 일괄 삭제 / CSV 내보내기

**Table Component**

| Column | Type | Sortable | Description |
|--------|------|----------|-------------|
| Checkbox | checkbox | ❌ | 행 선택 + 전체 선택 |
| ID | number | ✅ | 회원 고유 ID |
| 이름 | text | ✅ | 회원 실명 |
| 이메일 | text | ✅ | 로그인 이메일 |
| 전화번호 | text | ❌ | 연락처 |
| 상태 | badge | ✅ | Active / Suspended / Withdrawn |
| 보유 차량 수 | number | ✅ | 등록된 차량 대수 |
| 주문 횟수 | number | ✅ | 총 주문 건수 |
| 가입일 | datetime | ✅ | YYYY-MM-DD HH:mm |
| Actions | button | ❌ | 상세보기 / 수정 / 삭제 |

- 기본 정렬: 가입일 DESC
- 페이지네이션: 10 / 25 / 50 / 100건 per page
- 날짜 표시: YYYY-MM-DD HH:mm + 상대 시간 병기 (예: "3시간 전")

**Creation Modal — 회원 추가**

| Field | Type | Required | Validation | Error Message |
|-------|------|----------|------------|---------------|
| 이름 | text | ✅ | 2~50자, 특수문자 불가 | "이름은 2자 이상 50자 이하로 입력하세요" |
| 이메일 | email | ✅ | RFC 5321 이메일 형식, 중복 불가 | "올바른 이메일 형식이 아닙니다" / "이미 사용 중인 이메일입니다" |
| 전화번호 | tel | ✅ | 010-XXXX-XXXX 형식 (숫자+하이픈) | "올바른 전화번호 형식이 아닙니다 (010-0000-0000)" |
| 비밀번호 | password | ✅ | 8자 이상, 영문+숫자+특수문자 조합 | "비밀번호는 8자 이상, 영문/숫자/특수문자를 포함해야 합니다" |
| 비밀번호 확인 | password | ✅ | 위 비밀번호와 일치 | "비밀번호가 일치하지 않습니다" |
| 상태 | select | ✅ | Active / Suspended 선택 | "상태를 선택하세요" |

**Detail Drawer — 회원 상세**

표시 필드:
- 프로필 헤더: 이름, 이메일, 전화번호, 상태 badge
- 계정 정보: 가입일시, 최근 로그인, 로그인 유형
- 등록 차량 목록: 차량번호, 차종명, 등록일 (목록형)
- 주문 내역 요약: 최근 5건 (주문번호, 상품명, 금액, 상태, 주문일)
- 문의 내역 요약: 최근 3건 (제목, 상태, 작성일)
- 리뷰 내역: 작성 리뷰 수, 최근 리뷰 3건

가능한 액션:
- 회원 정보 수정 (이름, 전화번호)
- 상태 변경: 활성화 / 정지 (사유 입력 모달)
- 비밀번호 초기화 (임시 비밀번호 발송)
- 회원 삭제 (Soft Delete, 확인 다이얼로그)

**Delete Flow**
- Confirmation dialog: "홍길동 회원을 삭제하시겠습니까? 삭제된 회원 데이터는 30일간 보존 후 영구 삭제됩니다."
- 확인 클릭 시 Soft Delete (is_deleted = true, deleted_at 기록)
- 성공 토스트: "회원이 삭제되었습니다"
- Bulk 삭제 시: "선택한 N명의 회원을 삭제하시겠습니까?" 확인 다이얼로그

**Export**
- 버튼: "CSV 다운로드" / "Excel 다운로드"
- 내보내기 범위: 현재 필터/검색 결과 전체 (페이지 무관)
- 포함 필드: ID, 이름, 이메일, 전화번호, 상태, 가입일, 최근로그인, 주문횟수

**상태 화면**
- Loading: 테이블 행에 skeleton UI (10행)
- Empty: "등록된 회원이 없습니다" 안내 메시지
- Search No Results: "검색 결과가 없습니다" + "필터 초기화" 링크
- Error: "데이터를 불러올 수 없습니다" + Retry 버튼

**에러 핸들링**
- 네트워크 에러: 토스트 알림 + Retry 버튼
- 서버 에러 (500/503): "서버 오류" 안내 + Retry 버튼
- 권한 에러: 어드민 권한 없을 시 접근 차단

⚠️ **알려진 위험 (버그 패턴 기반)**
| # | 패턴 | 빈도 | 예방 스펙 |
|---|------|------|----------|
| 1 | 페이지네이션 미구현 및 정렬 오류 | 높음 (6회) | 목록 API는 page, page_size 파라미터 지원, 기본 정렬 created_at DESC 필수 구현 |
| 2 | 필터 적용 후 자동 초기화 | 높음 (5회) | 필터 조건을 URL query parameter에 저장, 뒤로가기 후 복원, 필터 초기화 버튼 별도 제공 |
| 3 | 관리자 테이블 Export 버튼 미동작 | 낮음 (2회) | Export 버튼은 API 연결 완료 전까지 비활성(disabled) 표시, onClick 핸들러 누락 금지 |

- 기본 정렬: 등록일 DESC
- 페이지네이션: 10 / 25 / 50 / 100건 per page

**Creation Modal — 장착점 추가**

- 포함 필드: ID, 장착점명, 대표자명, 사업자번호, 전화번호, 주소, 승인상태, 장착건수, 등록일

- Loading: 테이블 행 skeleton UI (10행)

- Search No Results: "검색 결과가 없습니다" + "필터 초기화" 링크

- 정산 금액 요약: 총 장착 건수, 합계 금액, 공제 금액, 실 정산 금액

- 취소는 상태 변경(정산취소)으로 처리

- 내보내기 범위: 현재 필터/검색 결과 전체
- 포함 필드: 정산번호, 장착점명, 정산기간, 장착건수, 정산금액, 정산상태, 정산처리일

### Export / Data Download

**Data Download — 공통 명세**

모든 어드민 관리 페이지에서 제공되는 Export 기능의 공통 동작 기준:

| 항목 | 명세 |
|------|------|
| 지원 형식 | CSV (.csv) / Excel (.xlsx) |
| 내보내기 범위 | 현재 필터/검색 조건이 적용된 전체 결과 (현재 페이지만이 아닌 전체 데이터) |
| 기간 필터 | 전체 기간 / 사용자 지정 날짜 범위 |
| 최대 건수 | 1회 최대 100,000건 (초과 시 "기간을 좁혀서 다시 시도하세요" 안내) |
| 파일명 규칙 | `{관리영역}_{YYYYMMDD}_{HHmmss}.csv` (예: `users_20260313_143000.csv`) |
| 인코딩 | UTF-8 BOM (Excel 한글 깨짐 방지) |
| 버튼 상태 | API 연결 완료 전까지 비활성(disabled) 표시 |
| 다운로드 방식 | 브라우저 파일 다운로드 트리거 (Blob URL) |

**관리페이지별 내보내기 가능 데이터**

| 관리 페이지 | CSV/Excel 다운로드 | 주요 포함 필드 |
|------------|-------------------|--------------|
| 회원 관리 | ✅ | ID, 이름, 이메일, 전화번호, 상태, 가입일, 주문횟수 |
| 장착점 관리 | ✅ | ID, 장착점명, 대표자명, 사업자번호, 주소, 승인상태, 등록일 |
| 상품 관리 | ✅ | ID, 상품명, 브랜드, 규격, 판매가, 재고, 생산년도/주차, 판매수 |
| 주문 관리 | ✅ | 주문번호, 회원명, 상품명, 결제금액, 배송유형, 주문상태, 주문일 |
| 정산 관리 | ✅ | 정산번호, 장착점명, 정산기간, 장착건수, 정산금액, 정산상태 |
| 이벤트 관리 | ✅ | ID, 이벤트명, 시작일, 종료일, 진행상태, 조회수 |
| 공지사항 관리 | ✅ | ID, 제목, 중요여부, 노출상태, 조회수, 등록일 |
| FAQ 관리 | ✅ | ID, 카테고리, 질문, 노출순서, 노출상태, 등록일 |
| 배너 관리 | ✅ | ID, 배너명, 노출위치, 시작일, 종료일, 클릭수 |
| 리뷰 관리 | ✅ | ID, 작성자명, 상품명, 평점, 노출상태, 등록일 |
| 문의 관리 | ✅ | ID, 문의유형, 제목, 작성자명, 답변상태, 등록일, 답변일 |

⚠️ **알려진 위험 (버그 패턴 기반)**

|---|------|------|----------|
